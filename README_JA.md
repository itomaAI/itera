# Itera OS

**ブラウザ内で完結する、自律型AIオペレーティングシステム**

Iteraは、**Host-Driven Intelligence (HDI)** アーキテクチャに基づき、AIエージェントがユーザーのブラウザ内環境（ファイルシステムやUI）を直接操作・構築・維持する実験的なOSです。

インストール不要で、ブラウザからすぐに起動できます。

> **注意:** 動作には Google Gemini API キーが必要です。APIキーはブラウザのローカルストレージ（localStorage）にのみ保存され、外部サーバーへ送信されることはありません。

## 💡 概要

従来のAIチャットボットは「会話」しかできませんでしたが、Iteraは**「行動」**します。
**REAL (Recursive Environment-Agent Loop)** アーキテクチャにより、AIは以下のループを自律的に回します。

1.  **観測 (Observe)**: 仮想ファイルシステム (VFS) と履歴を読み取る。
2.  **思考 (Think)**: 次に行うべき行動（コードの修正、ファイルの作成など）を計画する。
3.  **実行 (Act)**: ツールを実行して環境を書き換える。
4.  **更新 (Update)**: 結果をUIに即座に反映させる。

これにより、「ToDoアプリを作って」と頼めば、AIがその場でHTML/JSをコーディングし、ダッシュボードにアプリをデプロイします。

## ✨ 主な機能

### 1. 仮想ファイルシステム (VFS)
ブラウザのメモリと IndexedDB 上に構築された完全なファイルシステムです。
*   ファイル作成、編集、削除が可能。
*   ディレクトリ構造の維持。
*   ページをリロードしてもデータは永続化されます。

### 2. 再帰的な自己改善 (Recursive Self-Improvement)
IteraのUI（ダッシュボード）自体が VFS 上の `index.html` や `js/app.js` で構成されています。
つまり、**AI自身が自分自身のソースコードを書き換え、機能を拡張・修正することができます。**

### 3. タイムマシン (Time Machine)
AIがシステムを破壊することを恐れる必要はありません。
*   **スナップショット**: システム全体（ファイルと記憶）の状態をいつでも保存可能。
*   **リストア**: 何か問題が起きても、一瞬で過去の正常な状態に巻き戻せます。

### 4. 100% クライアントサイド実行
バックエンドサーバー（PythonやNode.js）は存在しません。
すべてのコード実行、ファイル操作、UIレンダリングは、あなたのブラウザの中で完結しています。

## 🛠️ アーキテクチャ

### Host (Core Layer)
*   **役割**: 脳、神の手。
*   **機能**: LLMとの通信、VFSの管理、ツールの実行権限を持つ特権領域。

### Guest (User Land)
*   **役割**: 身体、表示画面。
*   **機能**: サンドボックス化された `iframe` 内で動作。ユーザーが目にするダッシュボードやアプリはここで動く。
*   **Bridge**: `window.MetaOS` ライブラリを通じて、Hostに対してファイル操作やAIへの依頼を行う。

## 📦 使い方

1.  **起動**: [デモページ](https://itomaai.github.io/itera/) にアクセスします。
2.  **認証**: 右上の入力欄に Gemini API Key を入力し "Save" を押します。
3.  **対話**: チャット欄で指示を出します。
    *   例: 「タスク管理アプリを作って」
    *   例: 「背景をダークモードにして」
    *   例: 「dataフォルダの中に、今の時間を記録したテキストファイルを作って」
4.  **開発**: 必要に応じて、右側のファイルエクスプローラーから直接コードを編集することも可能です。

## ⚠️ トラブルシューティング

*   **画面が真っ白になった**: AIがコードを壊した可能性があります。サイドバーの「Time Machine (時計アイコン)」から以前のスナップショットに戻すか、「Factory Reset (赤いゴミ箱アイコン)」で初期化してください。
*   **応答が止まった**: "Stop" ボタンを押すか、ページをリロードしてください。状態はIndexedDBに保存されています。
